# Radzen Blazor MCP

A local [MCP](https://modelcontextprotocol.io) server that gives a coding agent
accurate Radzen Blazor component knowledge plus a markup-scaffolding tool. See
`docs/superpowers/specs/` for the design, `docs/adr/` for key decisions, and
`CONTEXT.md` for the project glossary.

## Layout

- `server/` — TypeScript/Node MCP server (stdio). Needs only Node 20+. **Built and tested.**
- `extractor/` — .NET tool that reflects `Radzen.Blazor` into `component-knowledge.json`. Needs the .NET 8 SDK. **Code-complete; not yet run (no .NET in the build env).**
- `component-knowledge.json` — committed knowledge base; the contract between extractor and server. **Generated by the extractor — see below.**

## Status

The server is complete and fully tested against `server/test/fixtures/knowledge.json`.
The `component-knowledge.json` knowledge base has not been generated yet because
the build environment has no .NET SDK. Until it exists, run the server against
the fixture via `RADZEN_KB_PATH` (below).

## Run the server

```sh
cd server
npm install
# Against the real knowledge base (once generated):
npm start
# Or against the test fixture, before the knowledge base exists:
RADZEN_KB_PATH=test/fixtures/knowledge.json npm start
```

## Test the server

```sh
cd server && npm test
```

## Tools

- `list_components(filter?)` — names + summaries, optional substring filter.
- `get_component(name)` — full API (parameters, events, summary).
- `search_components(query)` — match over component names, parameter names, descriptions.
- `scaffold_component(name, options)` — ready-to-paste markup, options validated against the component's parameters.

## Generate / regenerate the knowledge base

Requires the .NET 8 SDK and network access to restore the `Radzen.Blazor` package.

```sh
cd extractor && dotnet run -- ../component-knowledge.json
```

Re-run on a Radzen version bump and commit the updated `component-knowledge.json`.

## Register with an MCP client (example)

```json
{
  "mcpServers": {
    "radzen-blazor": {
      "command": "node",
      "args": ["--import", "tsx", "server/src/server.ts"]
    }
  }
}
```
